# MultiTool-Grupo3
Proyecto Final GitHub
Este proyecto tiene como finalidad mostrar el trabajo en equipo y el correcto uso de los repositorios remotos con los repositorios locales de cada integrante y una simulación de cómo trabajan los desarrolladores hoy en día y cómo pueden trabajar en equipo sin necesidad de estar juntos, destacando que Git es una herramienta versátil y eficaz.

El proyecto propone 5 mini-programas sencillos pero interesantes que se dividen de la siguiente manera:
ANALIZAR TEXTO, ADIVINA EL NÚMERO (Victor del Val, LÍDER)
CONVERSOR UNIDADES (Santiago Sarmiento, README)
GESTOR NÚMEROS (Nacho Enguidanos, dev)
GESTOR TAREAS "TEXTO" (Iván García, dev)

¿Qué hace el proyecto?
Vamos a pasar por cada desarrollador para explicar cómo funciona y su importancia en el proyecto.

1. ANALIZAR TEXTO, ADIVINA EL NÚMERO (Victor del Val, LÍDER)

Victor es nuestro líder del proyecto y se encargó de gran parte del mismo. Dio los permisos correspondientes a cada desarrollador, dividió las tareas y se encargó de organizar y dividir los espacios de trabajo para que cada uno hiciera su parte en la rama que le corresponde, para trabajar con un código más limpio y personal, de manera que ninguno "pisaba" el código del otro. Una vez que todos terminaron su código, él se encargó de unir cada parte.

//ANALIZAR TEXTO
Tiene un menú donde el programa te pedirá una frase y con ella podrás contar vocales, contar palabras y ver la palabra más larga de dicha frase. Si quieres cambiar de frase, solo debes dar a la opción cero e introducir otra frase.

//ADIVINA NÚMERO
Se va a generar aleatoriamente un número entre (1-50) y el usuario debe adivinar qué número es. Se te irá contando el número de veces que falles y si quieres volver a jugar podrás ver tus intentos de acierto en la partida anterior.

En el siguiente enlace tendrás acceso al código y commits realizados por el líder, podrás ver las ramas remotas. En ANALIZADOR DE TEXTOS tuvimos problemas para encontrar la palabra más larga porque no se recorría correctamente el array. Al ser un problema menor no hubo necesidad de documentarlo ni subir algún commit y en el ADIVINAR NÚMERO no se encontró dificultad alguna para desarrollar el código o realizar commits.

2. CONVERSOR UNIDADES (Santiago Sarmiento, README)

Santiago se encargó de documentar el proceso, commits y seguir dificultades de cada desarrollador. Durante el proceso tuvo inconvenientes en la realización de algunos commits, ya que subía un par de commits en la rama dev o main por confusión, ya que se pensaba que ya estaba en la rama correspondiente, pero el líder Victor se percataba y se encargaba de borrar dicho commit para hacerlo nuevamente.
